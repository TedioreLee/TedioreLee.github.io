🌈Migrate Azure VM to On-Premises

2019-04-24

A Step-by-step tutorial of Migrate Azure DC VM to On-Premises VM

Download VHD

在Azure Potal中，找到你的虚拟机并关闭

点击左侧的磁盘选项，下载整个虚拟机磁盘

PS: 建议使用例如idm这类的下载工具，浏览器自带下载可能会造成下载中断。

Add a new Domain Controller

打开Hyper-V管理器，新建一个用作之后本地主DC的虚拟机,配置自定，安装完成之后注意修改主机名，配置静态ip，为之后加域做好准备。

再新建一个虚拟机，配置注意保持与云端虚拟机一模一样防止启动报错，启动盘挂载为下载下来的vhd文件。

成功启动后，修改此虚拟机的ip以符合本地网络环境，注意关闭不必要的服务比如AAD Sync等，避免和云端起冲突。

在之前创建的DC中，将dns指向原云端DC，并加入域。

Promote system to a Domain Controller

在新DC中，添加角色，安装Active Directory Domain Services

等待安装完成之后，选择将此服务器提升为域控制器。

选择添加域控制器至现有的域

设置好你的DSRM密码

选择从旧的DC复制

等待检查安装并自动重启
